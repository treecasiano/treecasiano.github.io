Since yesterday afternoon I have been obsessively making small tweaks to this new Jekyll site. To install it, I used the helpful [Jekyll QuickStart Guide](http://jekyllbootstrap.com/usage/jekyll-quick-start.html#toc_0) on the Jekyll Boostrap site and found it to be a fairly straightforward process. I briefly ran into trouble with Ruby write permissions when I tried to install the jekyll gem, but I fixed that problem by installing Ruby with homebrew so that I'm no longer adding gems to the system Ruby.

It took a little while to get oriented to the structure of the files and how to make small changes to the default setting. Had I tried this six months ago when I was a very baby noob developer, I probably would have been put off by it. I'm nowhere near a Git Jedi now, but I'm proficient enough to not be daunted.

After I got everything up and running, I learned that there's a version of Jekyll specific to GitHub which requires you to use a special gem and a bundler to make sure all your dependencies stay current with GitHub's servers. Since everything is working so well now, I'm afraid to try to start over with something new. The worst case scenario is that I have to rebuild this site at some point, something I imagine would be only a minor inconvenience. No matter what what happens, I'll still have my asset and post files for a new verison.

All this happy Jekyll action has distracted me from the pre-work I'm supposed to be doing for an 8 hour Code Fellows Git & Unix workshop I'll be attending the day after tomorrow. If I'm inspired, I'll do a review of the training on this site. This workshop is a pre-requisite for the month-long Code Fellows JavaScript Foundations II course.
